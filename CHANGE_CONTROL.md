# Agent Hub 變更控制制度

此制度適用於 Codex、Claude Code、Cursor、人類開發者與任何自動化工具。目標是防止「修一處、無聲破壞另一條真實流程」。

## 每次變更前：五問確認

1. 這次要改善哪個使用者結果？用一句非技術語言寫出來。
2. 哪些 workflow、節點、資料、外部服務與觸發方式會受影響？先查 `DEPENDENCY_MAP.md`。
3. 是否會讀、寫、寄送、刪除、授權或暴露資料？若會，安全預設與確認點是什麼？
4. 可否在不碰真實資料下重現？不能時，如何用只讀或測試帳號驗證？
5. 如果失敗，版本還原與停止機制是否仍有效？

## 實作時：不可省略的閘門

- 改 workflow 結構、保存、執行、修復、授權、秘密、匯入／匯出、外部網址時，必須新增或更新自動測試。
- 改 UI 文案或流程時，必須以真實瀏覽器驗證：新手看不到不必要術語、關鍵按鈕可操作、窄畫面不跑版。
- 改 AI 相關程式時，必須同時檢查 builder、直接對話修改、點節點修復與 autorun；四條路的上下文與套用規則不能分裂。
- 改外部 I/O 時，必須驗證 cancel、timeout、dry-run、錯誤翻譯與不重複副作用。
- 修改完至少執行 `npm run lint && npx tsc --noEmit && npm test && git diff --check`；高風險改動還要跑受影響的真實瀏覽器／API 回歸。

## 變更紀錄格式

所有使用者可感知或高風險變更寫入 `CHANGELOG.md` 的 Unreleased 區塊：

`類型｜使用者問題｜改動範圍｜驗證證據｜未覆蓋風險／回復方式`

類型只能是 Added、Changed、Fixed、Security、Removed。不得只寫「優化」、「調整」或「修 bug」。

## 禁止事項

- 不得以整包舊 nodes 覆蓋最新 workflow。
- 不得因單一模型問題破壞共用重試層，或未實測視覺能力就把模型加入候選。
- 不得讓匯入流程保留外來可執行程式碼，或繞過首次信任確認。
- 不得把「成功沒有丟錯」當作「資料正確」。
- 不得把使用者卡住的情況丟回「請自己到某設定頁填欄位」；AI 必須能解釋目的、帶路，或明確說明不可自動完成的外部授權。

## 發版前人工驗收

至少選一條現有複雜流程與一條從零描述流程，驗收：附件理解、只讀測試、確認寫入、失敗定位、AI 修復、版本還原、停止執行。涉及 Google Slides 時另驗收 OAuth 只讀驗證與正式 batch refresh。
