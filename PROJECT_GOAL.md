# Agent Hub：已確認的產品目標

## 北極星

Agent Hub 是一個 **本機優先、以自然語言操作的高複雜工作流代理人**。它的使用者不必懂 API、程式、節點、模型、帳密格式或外部服務的技術細節；只要用口語、檔案、截圖、文件或網址描述工作，AI 就必須能理解實際資料與商業邏輯，建立、測試、執行、診斷並修好流程。

流程圖的角色是可確認、可追溯、可還原的工作說明，不是把使用者變成流程工程師。

## 產品承諾

（以下 1-6 項已於 2026-07-19 由使用者針對 Codex 步驟零提問逐項核准，取代 Codex 當初自行推斷的版本）

1. **零技術可用性、與高複雜跨系統可靠性，同等重要，不分先後**：不是先做簡單提醒版再往複雜擴充，也不是先打磨複雜案例才顧簡單案例——兩條線要並行推進。核心驗收既要涵蓋跨 Excel、Google Sheets、Google Slides、網頁資料、Email、PPT 與 LINE 的多步驟流程，也要讓完全不懂技術的人能獨立完成常見辦公流程。
2. **真的理解資料**：檔案、截圖、文件與網址必須送入合適的讀取／視覺／結構化解析路徑，不能只看檔名、OCR 片段或猜測。
3. **先讀、後寫**：AI 可以自行讀取、分析、診斷、重寫內部處理邏輯與只讀測試；外部寫入、寄送、刪除或發佈只能在使用者明確確認後執行。安全試跑仍必須真的跑到足以證明邏輯正確的位置。
4. **AI 必須修到有進展，且說明要簡短、看得懂**：修復可改上游、下游、結構與內部程式，不限紅色節點本身；每次修改都要能回溯（留版本／可還原），並用一句白話說清楚改了什麼、為什麼——不是大篇文字或技術性 log 堆疊，使用者要一眼看懂。沒有可行修法時，要立即指出真實缺口，不能重跑到逾時或假裝成功。
5. **外部服務由 AI 帶路**：OAuth、Google Cloud、LINE、帳密等必須由使用者本人授權，但介面要把操作翻成白話、可接收截圖、逐步引導，不要求使用者自行理解底層技術名詞。
6. **本機與單人優先，但不預先鎖死**：目前是私有單人本機產品，資料/帳密/執行紀錄留在使用者電腦。是否走多人協作／雲端部署尚未決定；在決定之前，不為了假設中的未來需求犧牲現在的簡單性與可靠性——不要提前引入多租戶／RBAC 之類的架構複雜度。

## 現階段優先驗收情境

使用者已明確表示：這份清單不能只固定在 Codex 當初提的三個範例，要持續納入更多真實情境——每次測試/除錯過程中發現的新情境類型，都應該回頭補進這份清單，而不是修完就忘。

- 信箱／下載檔 → Excel 計算 → Google Sheets 指定位置 → Google Slides 連結圖表刷新。
- 上傳或收取多份 Excel、PDF、Word、PPT、圖片 → 讀懂內容 → 比對、分析、產出報告或簡報。
- 上網找資料 → 驗證來源 → 分析 → 寄出 Email 或通知。
- 依自然語言修改既有流程；複製後只換目的地仍保留已驗證的商業邏輯。
- LINE Bot 作為一般人用口語啟動、查詢與接收結果的入口。
- **需要真人簽核的流程**：流程中途暫停，透過 Telegram／Email／桌面通知任一管道問真人，核准/拒絕各自接不同下游步驟(請假審核、大額支出複核、發文前確認)。
- **流程本身有「出錯時」備援分支**：某一步失敗但備援分支接手，讓整條 run 回報成功——即使如此，AI 仍必須主動講清楚「哪一步其實失敗了、為什麼」，不能因為備援接手就沉默。
- **需要一次性 OAuth／API 授權才能用的外部服務**：例如 Google Slides；使用者完全不熟技術名詞時，AI 要能全程白話帶著設定完成。
- **對清單批次重複同一組步驟**：例如處理連續好幾個月的報表，AI 建圖時要收成一份可重複定義，不能為了「每個月一份」而複製貼上一大串幾乎一樣的節點。
- **子流程組合**：把共用邏輯（例如「登入並下載報表」）抽成獨立流程，被多條主流程呼叫、結果接回主流程繼續。

## 已具備的基礎

- Next.js 本機介面、可縮放的流程畫布、對話、執行紀錄、版本備份與輸出檔案。
- JSON workflow 規格、節點 registry、圖結構 lint、增量儲存、原子備份與 SQLite 執行資料。
- builder／codegen／graph repair 三條 AI 路徑，以及檔案／網址／執行現場／失敗截圖上下文。
- 可取消的引擎、部分執行、乾跑、預覽確認、重試、排程、監聽、收信與訊息觸發。
- 本機帳密保護、匯入信任閘門、URL SSRF 防護、健康檢查、每日備份。
- Google Slides 官方 API 的 OAuth refresh 與 `refreshSheetsChart` 批次刷新能力；使用者完成一次授權後才可更新私人簡報。

## 「複製流程」的正式定義

複製全部可執行邏輯、節點設定、檔案脈絡與已確認規則，但**不**複製完整聊天內容、帳密、執行紀錄——避免舊對話污染新用途，也避免把帳密複製到不該有的地方。

## 非目標與不可假裝的能力

- 不會繞過 Google／Microsoft 的登入保護；使用者必須本人完成一次手動登入或 OAuth 授權。
- 不能在沒有來源、欄位對照或業務規則時猜測資料；此時必須問一個精準問題，或要求使用者上傳／授權讀取所需檔案。
- 不應以「全綠」當作完成；流程還必須通過資料傳遞、實際結果與使用者確認的語意驗收。
